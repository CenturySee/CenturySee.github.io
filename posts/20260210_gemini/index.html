躺在床上写代码？OpenClaw 让 Telegram 变成你的全栈 IDE

大家好，我是算法工程笔记。

还记得你上次被迫在假期打开电脑修 Bug 的痛苦吗？如果我告诉你，有人已经实现了“躺在床上，只用手机发发消息，就能完成全栈开发和发布”，这听起来是不是有点像科幻小说？

然而，就在前两天，avante.nvim 的作者 yetone 用实际行动证明了这不仅可行，而且效率惊人。他在休假期间，完全没有触碰电脑，仅通过 Telegram 和一个名为 OpenClaw 的 AI Agent，就完成了两个真实项目的迭代。

今天这篇，我们就来拆解一下这套“手机驱动全栈开发”背后的硬核逻辑。

01 当 Telegram 变成指挥中心#

这套玩法的核心，在于将**“指令”与“执行”**彻底分离。

简单来说，Telegram 是他的“嘴”，而 OpenClaw 是他的“手”。

  • 指挥官（Telegram）：作者巧妙地利用了 Telegram 的 Topic（话题） 功能。每一个 Topic 就是一个独立的“项目空间”。

    • Topic A 负责 macOS 桌面应用 Alma。
    • Topic B 负责 Web 服务 Confirmo。 这样做的好处是天然隔离了上下文，互不干扰，就像你在 IDE 里开了不同的 Workspaces。

  • 执行者（OpenClaw）：这是一个运行在服务器上的 AI Agent，但它远不止是一个聊天机器人。它拥有文件读写、Shell 执行甚至浏览器控制的权限。它不仅能写代码，还能编译、测试、部署。

02 真正闭环的工作流#

如果只是让 AI 写代码，那你还得自己复制粘贴、自己跑测试。OpenClaw 的强大之处在于它实现了 ChatOps 的完全闭环。

1. 需求下达 (Coding)#

你只需要在群里发一句人话：“给 Confirmo 加个 Telegram 通知功能。”

2. 自主编码与逻辑验证 (Logic Verification)#

OpenClaw 接收指令后，自动分析代码库并进行修改。

关键点来了：AI 写完代码后，会自动编写并运行单元测试。

如果测试挂了怎么办？Agent 会自己看日志、自己分析原因、自己修代码，然后重新跑测试。这个过程完全在服务器端自动循环，直到测试全部通过（Pass）。你不需要介入任何中间的报错环节，只需要等待结果。

3. 最暴力的视觉验收 (UI Verification)#

对于前端页面或者桌面应用，光代码跑通是不够的，界面丑了也不行。OpenClaw 的解决方案极其暴力且有效：截图。

  1. Agent 启动编译好的应用或网页。
  2. 调用系统截图工具，把运行画面截下来。
  3. 直接把截图发回 Telegram Topic。

作者拿着手机点开大图：“按钮颜色太浅了，改深一点。” Agent 收到反馈，改完再发新图，直到你回复“LGTM”（Looks Good To Me）。

4. 一键发布 (Deployment)#

验收通过后，Agent 直接执行 Git Commit，打 Tag，触发 CI/CD，应用自动上线。

03 从 Copilot 到 Autopilot#

为了证明这不是玩具，作者展示了两个截然不同的项目：

  • Alma：一个 macOS 原生桌面应用，涉及复杂的本地编译环境（Swift/Rust 等）。
  • Confirmo：一个 Web 服务，涉及后端逻辑、数据库和 Webhook 集成。

这给我们不仅仅是技术上的震撼，更是对未来开发模式的某种预示：

我们正在从“Writer（编写者）”转变为“Reviewer（审核者）”和“Decision Maker（决策者）”。

以前我们是和 Copilot 结对编程，现在我们是在管理 AI 员工。只要拥有执行环境（Terminal）和视觉反馈（Vision）能力，IDE 就不再是必需品，而只是一个选项。

以后的工位上，或许真的只需要一杯咖啡和一部手机了。
